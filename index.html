§ 00

摘要 Abstract

現代排球競技訓練對精確數據分析的需求日益迫切，然而傳統人工紀錄方式耗費大量人力、且容易產生主觀誤差。本研究提出一套整合電腦視覺與深度學習技術的排球比賽影像自動化分析系統，以單一固定視角的比賽錄影為輸入來源，透過物件偵測、球體追蹤、人體姿態估計，以及門控遞迴單元（Gated Recurrent Unit, GRU）時序模型，同步分析球體三維運動軌跡及球員動作序列。

系統可自動辨識舉球（Setting）、扣球（Spiking）、接球（Receiving）等技術動作，並輸出包括舉球軌跡分佈、攻擊成功率、球員移動熱區圖、每回合技術指標等結構化數據。本研究同時探討多種實驗方法——包含以 TrackNet 架構為核心的球體追蹤方案、以 MediaPipe BlazePose 結合 YOLOv8 的姿態估計方案，以及以 GRU 為時序預測骨幹的技術動作分類方案——並提供基準比較，協助後續研究者與教練依實際需求選擇最適策略。

§ 01

前言 Introduction

排球是一項高度仰賴團隊協作與技術精準度的競技運動。在高水平訓練中，每一次舉球的落點分佈、每一個扣球的成功率，都是指導訓練策略的關鍵依據。

1.1 研究背景與動機

近年來，運動科學已從傳統質性觀察走向量化數據驅動的訓練模式。職業籃球（NBA）、足球（英超）等聯賽廣泛引入電腦視覺系統進行球員追蹤與表現分析，顯著提升了戰術規劃的精確性。然而，相較之下，排球領域的自動化影像分析研究仍相對匱乏，尤其在以單一固定視角攝影機進行全場分析的場景下，技術挑戰更為顯著。

本研究聚焦於此一缺口：利用賽場已架設的固定攝影機錄影，無需額外昂貴的多鏡頭設備，即可自動化擷取球體軌跡、球員移動路徑、以及關鍵技術動作的姿態序列，最終輸出可供教練決策使用的統計數據。

1.2 研究問題

🎯
問題一：球體偵測困難
排球在比賽中移動速度快、影像模糊、時常被球員肢體遮擋，傳統目標偵測模型精度不足。
🏃
問題二：多人姿態估計干擾
場內 12 名球員交疊移動，如何精準追蹤特定位置（舉球手、接應）的姿態序列是一大挑戰。
🔢
問題三：時序動作分類
技術動作（舉球、扣球、攔網）涉及連續多幀的時序變化，需要有效的序列模型進行分類。

1.3 研究目的與貢獻

  • 建立一套可在固定視角單鏡頭下運行的排球比賽全場影像分析 pipeline。
  • 整合球體追蹤模組（TrackNet / YOLOv8）與骨架估計模組（MediaPipe / OpenPose），實現多任務同步分析。
  • 引入 GRU 時序模型，針對骨架關節序列進行技術動作分類，並與 LSTM、Transformer 方案進行基準比較。
  • 輸出實用的結構化球技數據報表，包含：舉球軌跡分佈圖、攻擊成功率統計、球員位置熱圖、每回合事件序列標記。
§ 02

相關研究 Related Work

2.1 運動影像中的物件偵測與球體追蹤

球體追蹤是運動影像分析中最具挑戰性的子問題之一。傳統方法依賴手工設計的特徵（如色彩直方圖、霍夫圓形偵測）對環境干擾高度敏感。深度學習方法的出現顯著提升了追蹤精度。

TrackNet（Huang et al., 2019）是此領域的代表性工作，其以 U-Net 架構為基礎，輸入連續多幀影像生成高斯熱圖（heatmap）定位球體，在網球廣播影片上達到 F1-measure 98.5%。TrackNet 的設計思路——透過連續幀學習軌跡動態而非單幀偵測——奠定了後續研究的重要基礎，亦可直接移植應用於排球場景。

WASB（Widely Applicable Strong Baseline, Tarashima et al., 2023） 進一步提出跨運動種類球體追蹤的通用基準架構，公開了對應資料集與程式碼庫，具備高度可復現性。

2.2 排球場景下的姿態估計

Liu et al. (2024) 在真實排球比賽場景中，結合 OpenPose 與 DeepSORT 完成即時多人姿態估計與追蹤，並使用 YOLOv4 進行人體偵測，驗證了此類組合在排球動態環境下的可行性。

Liu & Ikenaga (2024) 針對排球扣球動作提出動作感知（motion-aware）的多視角 3D 姿態精化方法，在標準資料集取得優於先前工作的精度。儘管本研究僅使用單鏡頭，其提出的關節修正策略仍具參考價值。

Google 的 MediaPipe BlazePose（Bazarevsky et al., 2020）提供了輕量、即時的 33 點全身骨架估計，適合作為資源有限場景下的姿態估計基礎，已廣泛用於各類運動分析研究。

2.3 排球群體行為辨識

Ibrahim et al. (CVPR 2016) 建立了著名的 Volleyball Dataset，提出階層式深度時序模型，以雙層 LSTM 分別捕捉個人動作動態與群體活動，成為後續排球影像分析研究引用最廣的資料集與基準。

2.4 GRU 在時序動作分類的應用

門控遞迴單元（GRU, Cho et al., 2014）作為 LSTM 的輕量替代方案，在較短序列的時序任務中往往達到相近甚至更好的效果，且訓練效率更高。在運動動作識別領域，以人體骨架關節序列為輸入的 GRU 模型已被廣泛驗證其有效性。

Wang et al. (2023) 以 CNN-GRU 架構對足球球員攻擊動作進行分析；Khobdeh & Yamaghani (2024) 則以深度模糊 LSTM/GRU 混合網路進行籃球動作辨識，均顯示了閘控機制在球類運動序列分析的廣泛適用性。

2.5 技術差距分析（Research Gap）

⚠ 本研究填補的空白
現有研究多聚焦於單一任務（僅追蹤球體、或僅進行動作識別），缺乏同時整合球體軌跡與球員姿態分析的端對端系統。此外，以固定視角單鏡頭針對舉球軌跡等排球特定技術指標進行自動化量化統計的研究幾乎空白，是本論文的核心創新點。

3.2 各模組說明

🎥
Module A：影片前處理
幀提取（FFmpeg）、亮度正規化、去模糊（可選）。偵測球場邊線，進行單應矩陣（Homography）投影轉換，將像素座標對應至真實場地座標。
⚪
Module B：球體追蹤
以 TrackNet V2（三幀輸入熱圖輸出）或 YOLOv8-small 為偵測核心，配合卡爾曼濾波器平滑化軌跡，輸出逐幀球體座標序列。
🏃
Module C：多人姿態估計
YOLOv8-pose 進行多人偵測與 17 點 COCO 關節估計，配合 DeepSORT 跨幀追蹤，對每位球員輸出逐幀骨架關節序列。
🧠
Module D：GRU 分類器
輸入連續 30 幀骨架特徵向量（關節座標 + 速度），以 2 層 Bidirectional GRU 輸出技術動作類別（舉球、扣球、發球、攔網、接球、其他）。
📊
Module E：事件整合
結合球體軌跡與動作分類結果，偵測每一次 rally 的起迄、技術動作序列、落點，計算舉球軌跡分佈、攻擊成功率等指標。
📋
Module F：輸出介面
輸出 CSV 數據表、軌跡標注視頻、球場熱區圖（使用 Matplotlib/Seaborn）、及 HTML 互動式報表。

3.3 GRU 模型詳細架構

# GRU 動作分類模型（PyTorch 偽代碼） class VolleyballActionGRU(nn.Module): def __init__(self, input_size=51, hidden_size=256, num_layers=2, num_classes=6): # input_size: 17 joints × (x, y, confidence) = 51 self.gru = nn.GRU( input_size=input_size, hidden_size=hidden_size, num_layers=num_layers, batch_first=True, bidirectional=True, # 雙向 GRU 捕捉前後文脈 dropout=0.3 ) self.attention = SelfAttention(hidden_size * 2) # 注意力機制 self.fc = nn.Linear(hidden_size * 2, num_classes) self.dropout = nn.Dropout(0.4) def forward(self, x): # x shape: [batch, seq_len=30, input_size=51] out, _ = self.gru(x) # [batch, 30, 512] out = self.attention(out) # [batch, 512]（加權平均） out = self.dropout(out) return self.fc(out) # [batch, 6] # 輸入特徵向量設計 features = [ joint_coords, # 17 × 2 = 34 維 joint_velocity, # 17 × 2 = 34 維（相鄰幀差） joint_conf, # 17 × 1 = 17 維（估計信心分數） ] # 合計 85 維輸入特徵
§ 04

實驗方法 Experimental Methods

本研究提供三種核心模組的複數實驗方案，供選擇與比較。每個模組均可獨立替換，方便後續依資源、精度需求調整。

METHOD A 球體偵測與軌跡追蹤方案比較 ▼

針對排球球體的偵測與軌跡追蹤，本研究比較以下三種方案：

方案 核心模型 架構特點 優點 限制 推薦場景
A-1
推薦優先		 TrackNet V2 U-Net + 三幀輸入高斯熱圖 專為高速小物體設計；對模糊、部分遮擋魯棒；軌跡連貫性高 需自行標記排球訓練資料 排球廣播視角、有遮擋的場景
A-2 YOLOv8-small + ByteTrack 單幀偵測 + 多目標追蹤 訓練資料充足（Roboflow Universe）；部署靈活；速度快 球體模糊時偵測率下降 高解析度、球體清晰的場景
A-3 WASB（Tarashima et al.） 多尺度 FCN + 跨運動種類泛化 公開程式碼與資料集；可直接 fine-tune 排球特化性低於 A-1 初期快速驗證、多運動種類比較
✅ 建議策略
以 A-1（TrackNet V2）為主要方案，標記約 500～1000 幀排球影片進行 fine-tune；以 A-2 作為 Baseline 對照實驗；以卡爾曼濾波器 + Savitzky-Golay 濾波進行軌跡平滑化後處理。
METHOD B 球員姿態估計方案比較 ▼

針對多球員骨架估計，本研究比較三種主流框架：

方案 工具 關節點數 速度（RTX 3060） 特點
B-1
推薦		 YOLOv8-pose + DeepSORT 17 點（COCO） ~30 fps 偵測追蹤整合；跨幀 ID 穩定；Ultralytics 官方支援
B-2 MediaPipe BlazePose 33 點（全身） ~25 fps（CPU 可用） 輕量部署；單人精度高；手指、臉部關節豐富；但多人需配合 YOLOv8 裁切
B-3 OpenPose + DeepSORT 25 點（BODY_25） ~15 fps 成熟框架；大量文獻採用；速度較慢
💡 推薦組合
B-1（YOLOv8-pose） 做多人追蹤主模型，對舉球手等關鍵球員裁切後額外以 B-2（MediaPipe BlazePose） 進行精細骨架分析，以獲取更豐富的上肢關節資訊（手腕、手肘角度對舉球姿態分析至關重要）。
METHOD C 時序動作分類模型比較（GRU vs. 其他） ▼

以骨架關節序列（連續 30 幀）為輸入，比較四種時序建模策略：

方案 模型 參數量 訓練收斂速度 動作分類精度（預估）
C-1
本研究核心		 Bi-GRU + Attention ~1.2M 快（梯度流動效率高） 88~92%
C-2 Bi-LSTM + Attention ~1.8M 中 87~91%
C-3 Transformer（Temporal） ~4M 慢（需較多資料） 89~93%（資料充足時）
C-4 1D-CNN + GRU（混合） ~0.8M 快 85~89%
⚠ 資料標記建議
建議標記至少 500 個動作片段（每類約 80 個），涵蓋不同球員、不同光線條件、正面與側面視角，以確保分類模型的泛化能力。可利用 CVAT 或 Label Studio 進行標記。

輸入特徵向量設計選項：

  • 特徵組 F1（基礎）：17 關節 (x, y) 座標 + 信心分 → 51 維
  • 特徵組 F2（加速度）：F1 + 逐幀速度向量 → 85 維
  • 特徵組 F3（角度）：F2 + 12 個關節角度特徵（肩、肘、腕、髖、膝、踝）→ 97 維（推薦）
  • 特徵組 F4（含球位置）：F3 + 球體相對座標 → 99 維（舉球分析最佳）

4.1 損失函數與訓練策略

🎯 分類任務
採用 Focal Loss（γ=2.0）應對類別不平衡（舉球動作片段數量可能遠多於攔網），配合 Label Smoothing（ε=0.1）防止過擬合。
🔧 訓練技巧
學習率使用 Cosine Annealing（初始 1e-3），配合 Early Stopping（patience=10）。資料增強包含水平翻轉座標鏡像、時序速度微擾（±10%）、隨機遮擋部分關節。
§ 05

資料集 Datasets

本研究使用公開資料集進行模型預訓練與基準測試，並自行錄製比賽影片進行 fine-tune 與最終評估。

🏐
Volleyball Dataset（Ibrahim et al., CVPR 2016）
最廣泛引用的排球群體行為辨識資料集。包含 55 段廣播影片（4830 幀），每幀標記 8 種個人動作類別及 8 種群體活動類別，含球員框（bounding box）標記。適合用於球員動作辨識模型的預訓練。
🔗 https://github.com/mostafa-saad/deep-activity-rec
🎾
TrackNet Tennis Dataset（NCTU, 2018）
TrackNet 原始訓練資料集。包含 10 段廣播網球影片、共 20,844 個標記幀，每幀含球體位置（X, Y）、可見度分類與軌跡樣式。球體追蹤模型的 Pre-training 極佳來源，可遷移至排球域。
🔗 https://drive.google.com/open?id=1GzJZeEPEi8lJjEAAtVnHhRdX8TVR14yK
⚽
WASB（Widely Applicable Strong Baseline for SBDT, 2023）
跨運動種類球體偵測與追蹤基準資料集，涵蓋多種球類運動的標記影像序列，包含完整程式碼庫，可直接進行模型對比實驗。
🔗 ResearchGate: WASB Paper
🏐
Volleyball Ball Object Detection Dataset（Roboflow / Kaggle）
Kaggle 上公開的排球球體標記資料集，含逐幀 bounding box 標記，適合作為 YOLOv8 球體偵測的訓練資料。可作為 A-2 方案的訓練基礎。
🔗 https://www.kaggle.com/datasets/pythonistasamurai/volleyball-ball-object-detection-dataset
🏃
Fast Volleyball Tracking Dataset（GitHub: asigatchov）
開源排球球體追蹤推論系統，基於 ONNX 優化模型，在 CPU 上可達 100 FPS，輸出球體座標至 CSV，含完整的 Rally 切割工具。可用於快速原型驗證。
🔗 https://github.com/asigatchov/fast-volleyball-tracking-inference
📹
自行錄製資料集（本研究最重要來源）
建議與當地大學或體育系排球隊合作，以固定三腳架 + 廣角攝影機錄製完整比賽或訓練賽（建議至少 10 場，每場 3 局）。標記工具可使用 CVAT（cvat.ai）或 Roboflow。重點標記：球體位置、每位球員 ID、技術動作類型（舉球/扣球/接球/攔網/發球）。
🔗 標記工具：https://www.cvat.ai/
